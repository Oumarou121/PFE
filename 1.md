# 📋 Plan des Sprints - SIRH Doc Editor

## 🎯 Vue d'ensemble

**Durée totale**: 8 semaines (4 sprints de 2 semaines chacun)
**Équipe**: 1 développeur full-stack
**Objectif**: Développer une plateforme complète de gestion et d'édition de documents avec authentification et contrôle d'accès

---

# 📅 SPRINT 1 : Fondations & Infrastructure (Semaines 1-2)

## 🎯 Objectif Sprint

Mettre en place l'infrastructure complète du projet (backend, database, authentification JWT)

## 📝 User Stories

### US1.1: Configurer l'architecture backend

- **Description**: Initialiser le projet ASP.NET Core 8, structure des couches
- **Tâches**:
  - Créer la structure de dossiers (Controllers, Services, Repositories, Domain)
  - Installer les dépendances Dapper, SQL Client
  - Configurer la base de données SQL Server
  - Configurer le logging et les middlewares

### US1.2: Implémenter l'authentification JWT

- **Description**: Créer le système d'authentification par JWT Bearer
- **Tâches**:
  - Implémenter l'entité User
  - Créer AuthController (Login, Register)
  - Configurer JWT (token generation, validation)
  - Ajouter BCrypt pour le hachage des mots de passe
  - Implémenter UserRepository

### US1.3: Configurer la base de données

- **Description**: Créer le schéma et les tables principales
- **Tâches**:
  - Créer la table Users avec les champs requis
  - Ajouter les contraintes et indexes
  - Créer les scripts de migration
  - Implémenter EditorDbConnectionFactory

### US1.4: Mettre en place Swagger/OpenAPI

- **Description**: Documenter l'API avec Swagger
- **Tâches**:
  - Configurer Swagger
  - Ajouter la sécurité JWT à Swagger
  - Documenter les endpoints d'authentification

### US1.5: Configurer CORS et la sécurité

- **Description**: Configurer les politiques CORS et les headers de sécurité
- **Tâches**:
  - Configurer la politique CORS "EditorCors"
  - Ajouter les headers de sécurité (HSTS, X-Frame-Options)
  - Tester les requêtes cross-origin

## 📦 Livrables

- ✅ Backend fonctionnel avec authentification JWT
- ✅ Base de données SQL Server avec schéma de base
- ✅ Swagger accessible et documenté
- ✅ Tests d'authentification réussis

---

# 📅 SPRINT 2 : Fonctionnalités de Base (Semaines 3-4)

## 🎯 Objectif Sprint

Développer les fonctionnalités métier principales et le système d'accès par rôles

## 📝 User Stories

### US2.1: Implémenter le système de rôles (RBAC)

- **Description**: Créer le contrôle d'accès basé sur les rôles (supAdmin, admin, user)
- **Tâches**:
  - Ajouter le champ Role à l'entité User
  - Implémenter les rôles (supAdmin, admin, user)
  - Créer les policies d'autorisation
  - Ajouter l'attribut [Authorize(Roles="...")] aux contrôleurs

### US2.2: Gérer les entités métier (Families, Templates, GraphicCharters)

- **Description**: Créer les entités et repositories pour les données métier
- **Tâches**:
  - Créer les entités Domain (Family, Template, GraphicCharter, EditorUser)
  - Implémenter les repositories (EditorRepository)
  - Créer les DTOs pour la sérialisation
  - Ajouter les tables en base de données

### US2.3: Créer les contrôleurs métier

- **Description**: Implémenter les endpoints pour les opérations CRUD
- **Tâches**:
  - Créer EditorController (CRUD documents)
  - Créer FamiliesController
  - Créer TemplatesController
  - Créer GraphicChartersController
  - Implémenter la logique métier dans EditorService

### US2.4: Configuration du profil utilisateur

- **Description**: Permettre aux utilisateurs de gérer leurs profils
- **Tâches**:
  - Créer le endpoint GET /api/auth/profile
  - Ajouter les champs Profile et ProfileDetail
  - Implémenter la modification du profil
  - Créer TableViewConfigsController pour les préférences UI

### US2.5: Validation des données

- **Description**: Valider les requêtes entrantes
- **Tâches**:
  - Ajouter la validation dans les DTOs
  - Implémenter les validations métier
  - Ajouter les messages d'erreur personnalisés

## 📦 Livrables

- ✅ Système de rôles et autorisation fonctionnel
- ✅ Endpoints CRUD pour toutes les entités métier
- ✅ Base de données complète avec tous les schémas
- ✅ DTOs et validation des données
- ✅ Tests manuels des endpoints via Swagger

# 📅 SPRINT 3 : Frontend & Interface Utilisateur (Semaines 5-6)

## 🎯 Objectif Sprint

Développer l'interface Angular avec l'éditeur riche de texte et la navigation par rôles

## 📝 User Stories

### US3.1: Initialiser le projet Angular et structure

- **Description**: Configurer le projet Angular 17 et l'architecture modulaire
- **Tâches**:
  - Créer la structure de dossiers (core, features, shared, layout)
  - Configurer le routing avec chargement lazy
  - Installer Angular Material
  - Configurer TypeScript et les styles SCSS

### US3.2: Implémenter l'authentification frontend

- **Description**: Créer le système de login et gestion des tokens
- **Tâches**:
  - Créer le composant LoginComponent
  - Implémenter AuthService (appels à l'API)
  - Ajouter les guards (loginGuard, roleGuard)
  - Implémenter le stockage des tokens (localStorage)
  - Créer l'interceptor pour les requêtes JWT

### US3.3: Créer la navigation par rôles

- **Description**: Développer les interfaces selon les rôles utilisateur
- **Tâches**:
  - Créer SuperAdminComponent
  - Créer AdminComponent
  - Créer UserPageComponent
  - Implémenter la barre de navigation adaptée aux rôles
  - Ajouter le logout

### US3.4: Intégrer Tiptap (Rich Text Editor)

- **Description**: Implémenter l'éditeur riche avec Tiptap
- **Tâches**:
  - Installer Tiptap et les extensions
  - Créer un composant EditorComponent réutilisable
  - Ajouter les extensions (tables, images, couleurs, fonts)
  - Implémenter la barre d'outils
  - Gérer la sauvegarde du contenu

### US3.5: Créer les interfaces de gestion

- **Description**: Développer les pages de gestion des données
- **Tâches**:
  - Créer les composants pour Families
  - Créer les composants pour Templates
  - Créer les composants pour GraphicCharters
  - Ajouter les listes avec pagination
  - Implémenter les formulaires de création/édition

### US3.6: Mise en page et design

- **Description**: Appliquer les styles et chartes graphiques
- **Tâches**:
  - Créer le layout principal (header, sidebar)
  - Appliquer les styles SCSS
  - Implémenter le responsive design
  - Ajouter les icônes et images

## 📦 Livrables

- ✅ Application Angular 17 fonctionnelle
- ✅ Système d'authentification et de rôles
- ✅ Éditeur riche Tiptap avec barres d'outils
- ✅ Interfaces de gestion pour toutes les entités
- ✅ Design responsive et intégration Material

# 📅 SPRINT 4 : Intégration, Tests & Polissage (Semaines 7-8)

## 🎯 Objectif Sprint

Intégrer le backend et frontend, réaliser les tests, optimiser et préparer le déploiement

## 📝 User Stories

### US4.1: Intégration complète Backend-Frontend

- **Description**: Vérifier que tous les appels API fonctionnent correctement
- **Tâches**:
  - Tester tous les endpoints avec le frontend
  - Corriger les problèmes d'intégration
  - Vérifier les erreurs et les réponses API
  - Adapter les DTOs si nécessaire

### US4.2: Tests unitaires et intégration

- **Description**: Créer des tests pour garantir la qualité
- **Tâches**:
  - Créer des tests unitaires pour AuthService
  - Créer des tests d'intégration pour les endpoints critiques
  - Créer des tests pour les repositories
  - Atteindre au moins 70% de couverture

### US4.3: Sécurité et optimisation

- **Description**: Renforcer la sécurité et les performances
- **Tâches**:
  - Sécuriser les secrets (déplacer en env vars)
  - Ajouter rate limiting
  - Ajouter le logging centralisé
  - Optimiser les requêtes BD (indexes, N+1 queries)
  - Ajouter la pagination si absente

### US4.4: Gestion des erreurs et validation

- **Description**: Améliorer la gestion des erreurs
- **Tâches**:
  - Ajouter les validations FluentValidation (backend)
  - Créer un error handler global
  - Ajouter les messages d'erreur au frontend
  - Implémenter retry logic si nécessaire

### US4.5: Documentation et déploiement

- **Description**: Documenter le projet et préparer le déploiement
- **Tâches**:
  - Créer le README avec instructions de setup
  - Documenter les API dans Swagger
  - Créer un Dockerfile pour chaque service
  - Créer docker-compose.yml pour orchestration
  - Créer un script de déploiement

### US4.6: Performance et optimisation frontend

- **Description**: Optimiser l'application Angular
- **Tâches**:
  - Ajouter le lazy loading pour les modules
  - Optimiser les images et assets
  - Ajouter le caching côté client
  - Tester les performances avec Chrome DevTools
  - Optimiser le bundle size

### US4.7: Rapport PFE et présentation

- **Description**: Finaliser la documentation pour le PFE
- **Tâches**:
  - Écrire le rapport technique
  - Créer les diagrammes d'architecture
  - Documenter les choix technologiques
  - Préparer la présentation orale
  - Créer des screenshots et démos

## 📦 Livrables

- ✅ Application complètement intégrée et testée
- ✅ Suite de tests avec bonne couverture
- ✅ Code sécurisé et optimisé
- ✅ Docker et scripts de déploiement
- ✅ Documentation complète
- ✅ Rapport PFE finalisé

## 💡 Conseils Supplémentaires

1. **Daily Standup**: Même si vous êtes seul, documentez quotidiennement ce que vous faites
2. **Git**: Committez régulièrement avec des messages clairs
3. **Code Review**: Relisez votre code à la fin de chaque tâche
4. **Documentation**: Documentez en même temps que vous codez
5. **Tests**: Écrivez les tests pendant le développement, pas à la fin
